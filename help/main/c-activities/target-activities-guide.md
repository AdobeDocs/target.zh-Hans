## 该活动有什么作用？ {#section_4ECAACC68723402EB3649033190E1BBC}

| 活动类型 | 详细信息 |
|--- |--- |
| 手动 [!UICONTROL A/B 测试]<P>![图标](/help/main/c-activities/assets/icon_ab.png) | 比较两个或更多体验，以查看在预先指定的一段测试时间内，哪个体验最有利于提高转化。<P>有关更多信息，请参阅 [A/B 测试](/help/main/c-activities/t-test-ab/test-ab.md)。 |
| [!UICONTROL 自动分配]<P>![自动分配图标](/help/main/c-activities/assets/icon_auto_allocate.png) | 从两个或更多体验中确定入选者，然后将流量重定向到入选体验，随着测试的运行和学习而提高转化。<P>有关更多信息，请参阅[自动分配](/help/main/c-activities/automated-traffic-allocation/automated-traffic-allocation.md)。 |
| [!UICONTROL 自动定位]<P>![AT 图标](/help/main/c-activities/assets/icon_auto_target.png) | 采用先进的机器学习技术，确定多个高性能、由营销人员定义的体验，然后根据访客各自的客户配置文件和过去类似访客的行为，为其提供量身定制的体验，从而实现内容的个性化并推动转化。<P>有关更多信息，请参阅[自动定位以提供个性化体验](/help/main/c-activities/auto-target/auto-target-to-optimize.md)。 |
| [!UICONTROL Automated Personalization] (AP)<P>![AP 图标](/help/main/c-activities/assets/icon_ap.png) | 采用先进的机器学习技术，结合特定选件或消息，然后根据访客各自的客户配置文件将不同的选件变体与其进行匹配，从而实现内容的个性化并推动转化。<P>有关更多信息，请参阅 [Automated Personalization](/help/main/c-activities/t-automated-personalization/automated-personalization.md)。 |
| [!UICONTROL 多变量测试] (MVT)<P>![MVT 图标](/help/main/c-activities/assets/icon_mvt.png) | 比较页面上元素之间的选件组合，看哪个选件组合对于特定受众表现最佳。此外，在预先指定的整个测试期间，确定该页面的哪个元素最有利于提高转化。<P>有关更多信息，请参阅[多变量测试](/help/main/c-activities/c-multivariate-testing/multivariate-testing.md)。 |
| 体验定位 (XT)<P>![XT 图标](/help/main/c-activities/assets/icon_xt.png) | 根据营销人员定义的一套规则和标准向特定受众投放内容。<P>有关更多信息，请参阅[体验定位](/help/main/c-activities/t-experience-target/experience-target.md)。 |

## 您为什么使用此类活动？ {#section_46A70DD7CE3448749E635DDF5EAFC131}

| 活动类型 | 原因 |
|--- |--- |
| 手动 [!UICONTROL A/B 测试] | 高度可控的流量测量实验，按百分比而不是按规则进行拆分，让您能够分析测试数据、收集有关受众的分析，并确定性能最佳的体验。 |
| [!UICONTROL 自动分配] | 确定入选体验并调整流量分配以尽可能快地将入选体验传递给访客，支持更快、更高可能性地进行转化。 |
| [!UICONTROL 自动定位] | 在多个体验中确定入选者，然后向特定访客提供最适合的体验。随着访客兴趣的不断变化，定位会随着时间的推移而进行调整，因为该算法可以预测访客在特定时间对某种体验的转化倾向。 |
| [!UICONTROL Automated Personalization] (AP) | 该方法可个性化一组选件（创建或预定义，在单个页面上或多个页面上的元素中），并交付最能吸引特定访客的选件组合。 |
| [!UICONTROL Multivariate Testing] (MVT) | 该方法可在多个元素中显示多个选件，然后针对特定目标同时对所获得的独特体验进行测试，这有助于确定哪个元素变体最为成功。MVT 活动还可以揭示哪些元素对访客的交互可产生最大的正面或负面影响。 |
| [!UICONTROL 体验定位] (XT) | 一种根据一组定义的分配规则将特定内容定位到特定受众的方法。 |

## 哪些类型的营销人员应使用哪种活动？ {#section_A843D663D3E543FFB1A594266B560395}

| 活动类型 | 营销人员 |
|--- |--- |
| 手动 [!UICONTROL A/B 测试] | 熟知相关统计数据知识。<P>具有足够的时间等待测试期结束来分析结果。 |
| [!UICONTROL 自动分配] | 时间范围较短。<P>必须确定最佳体验并快速投放。<P>希望能够在测试运行时“偷看”结果。 |
| [!UICONTROL 自动定位] | 拥有几个符合条件的体验。<P>想要根据其动态和不断变化的配置文件，在最佳时间将体验与特定访客匹配。 |
| [!UICONTROL Automated Personalization] (AP) | 具有一个或多个选件。<P>想要创建选件组合，以便通过各种唯一的配置文件和行为来为特定访客提供最佳的个性化体验。 |
| [!UICONTROL Multivariate Testing] (MVT) | 熟知相关统计数据知识。<P>具有一个或多个选件。<P>想要分析与页面元素交互相关的转化趋势。 |
| [!UICONTROL 体验定位] (XT) | 必须向特定受众投放特定体验或内容。 |

## 统计详细信息 {#section_22CF2D07DB054505AB5EC702B99A5BB0}

| 活动类型 | 详细信息 |
|--- |--- |
| 手动 [!UICONTROL A/B 测试] | 该测试将每个挑战者的体验与控制体验进行比较，然后对所有体验的性能进行排名，从而确定与控制体验相比的入选体验和落选体验。 |
| [!UICONTROL 自动分配] | 该测试能够立即产生真正的入选者并在统计上予以保证，然后将更多流量引导至可能会在该入选体验具有更高转化可能性的受众。 |
| [!UICONTROL 自动定位] | 该优化机制通过显示提升度在一段时间内的增加和降低情况来确定每个体验的相关受众，并在之前确定向哪个访客提供哪个体验。该优化机制参考转化、区段、参数和配置文件脚本。此时，该机制会自动选择使用哪种算法，以便产生更高的提升率和转化率。 |
| [!UICONTROL Automated Personalization] (AP) | 根据新访客和过去类似访客的行为，该优化机制会不断调整向哪些访客传递哪些体验，并根据并发的控制组来测量选件的性能。 |
| [!UICONTROL Multivariate Testing] (MVT) | 该测试有助于揭示特定元素对转化的相对影响。 |
| [!UICONTROL 体验定位] (XT) | 该方法定义了将特定体验或特定内容定位到特定受众的规则。客户可以在体验级别进行更新。 |

## 优点和注意事项 {#section_56C46ABEF7B945DDA0C1E6D714377123}

| 活动类型 | 优点 | 注意事项 |
|--- |--- |--- |
| 手动 [!UICONTROL A/B 测试] | A/B 测试可帮助您全面了解每个体验的执行方式，而不仅仅是哪个体验性能最佳。 | 在 [!UICONTROL A/B 测试]中，如果您在达到样本量之前查看测试结果，则可能存在这样的风险：您依赖的结果并不准确（您不能提前“偷看”！）。<P>与[!UICONTROL 自动分配]不同，在 A/B 测试中，即使在您了解到某些体验优于其他体验之后，A/B 测试的流量分配仍保持不变。<P>有关 [!UICONTROL A/B 测试]活动的最佳实践的信息，请参阅 [A/B 测试应该持续多长时间](/help/main/c-activities/t-test-ab/sample-size-determination.md)和[十个常见的 A/B 测试隐患以及避免方法](/help/main/c-activities/t-test-ab/common-ab-testing-pitfalls.md)。 |
| [!UICONTROL 自动分配] | [!UICONTROL 自动分配减少了典型 A/B 测试的成本，因为它具有比手动 A/B 测试更高的整体转化率。]转化率更高是因为[!UICONTROL 自动分配]将更多流量推送到具有最高性能的体验，这意味着您可以在测试期结束之前提前了解入选体验的优点（可以“偷看”！）。 | [!UICONTROL 自动分配]会识别入选者，但不区分落选者。如果您必须了解每个体验的性能，那么 A/B 测试更适合。<P>[!UICONTROL 自动分配]功能只能与一项高级量度设置配合使用，即“递增计数并保持用户处于活动中”。如果您不想计算重复的转化次数，则应改用 A/B 测试。 |
| [!UICONTROL 自动定位] | 使用[!UICONTROL 自动定位]时，机器学习可以应用于任何类型的体验，包括多页面体验。[!UICONTROL 自动定位]活动还可让您在使用熟悉的 A/B 测试工作流的同时，获得 [!UICONTROL Automated Personalization] 的好处。 | 使用[!UICONTROL 自动定位]时，如果您想频繁地更改选件内容，则在每次更改后，该算法都需要足够的时间才能充分利用所学内容，并将该内容真正交付到合适的访客。 |
| [!UICONTROL Automated Personalization] (AP) | 通过 [!UICONTROL Automated Personalization]，您可以将所有选件收集到一个位置，该算法将确定选件的最佳组合。您无需指定或构建每一个体验。[!UICONTROL Automated Personalization] 使用与[!UICONTROL 自动定位]相同的机器学习算法。 | 当您合并多个选件时，会发生组合爆炸，导致需要大量的流量。[!UICONTROL Automated Personalization] 算法将多个因素考虑在内；因此，需要的流量最多。<P>[!UICONTROL Automated Personalization] 无法使用 [Analytics for Target](/help/main/c-integrating-target-with-mac/a4t/a4t.md) (A4T) 中的报告。 |
| [!UICONTROL Multivariate Testing] (MVT) | 通过 [!UICONTROL Multivariate Testing]，您可以同时测试多个元素。 | [!UICONTROL Multivariate Test] 非常耗时，并且由于存在多个变量，因此不一定会产生具有置信度的入选体验。<P>达到完成这种测试所需的流量通常是一项困难的挑战。由于所有[!UICONTROL 多变量测试]都是全因子实验，因此，在同一时间可能会有太多的变化元素，从而导致必须测试的可能组合会增大到一个较大的数量。<P>即使是流量相当高的网站也可能无法在可行的时间内完成超过 25 种组合的测试。 |
| [!UICONTROL 体验定位] (XT) | 通过[!UICONTROL 体验定位]，您可以针对从任何活动结果推断出的分析进行快速行动。<P>例如，如果您进行了 A/B 测试，测试结果表明，挑战体验的表现并未超出控制体验，但与控制体验相比，访客的一个非常特定的区段实际上在挑战体验上的转化率高出了 4 倍多，那么此时，您可以使用[!UICONTROL 体验定位]将挑战体验引导至该特定区段。 | [!UICONTROL 体验定位]不允许您控制多个受众在一个体验中的比例。 |
