>
>为了获得最佳体验并与他人共享，请下载交互式 [Adobe Target 活动指南 PDF](/help/main/assets/activities_guide_82817.pdf)。

## 该活动有什么作用？ {#section_4ECAACC68723402EB3649033190E1BBC}

| 活动类型 | 详细信息 |
|--- |--- |
| 手动 [!UICONTROL A/B测试]<P>![图标](/help/main/c-activities/assets/icon_ab.png) | 比较两个或更多体验，以查看在预先指定的测试期间，哪个体验最有利于提高转化。<P>有关更多信息，请参阅 [A/B 测试](/help/main/c-activities/t-test-ab/test-ab.md)。 |
| [!UICONTROL 自动分配]<P>![自动分配图标](/help/main/c-activities/assets/icon_auto_allocate.png) | 从两个或更多体验中标识一个入选者，然后将流量重定向到入选体验，随着测试的运行和学习而提高转化。<P>有关更多信息，请参阅[自动分配](/help/main/c-activities/automated-traffic-allocation/automated-traffic-allocation.md)。 |
| [!UICONTROL 自动定位]<P>![图标在](/help/main/c-activities/assets/icon_auto_target.png) | 采用先进的机器学习技术，确定多个高性能、由营销人员定义的体验，然后根据访客各自的客户配置文件和过去类似访客的行为，为其提供量身定制的体验，从而实现内容的个性化并推动转化。<P>有关更多信息，请参阅[自动定位以提供个性化体验](/help/main/c-activities/auto-target/auto-target-to-optimize.md)。 |
| [!UICONTROL 自当个性化] (AP)<P>![图标AP](/help/main/c-activities/assets/icon_ap.png) | 采用先进的机器学习技术，结合特定选件或消息，然后根据访客各自的客户配置文件将不同的选件变体与其进行匹配，从而实现内容的个性化并推动转化。<P>有关更多信息，请参阅[自动个性化](/help/main/c-activities/t-automated-personalization/automated-personalization.md)。 |
| [!UICONTROL 多变量测试] (MVT)<P>![MVT图标](/help/main/c-activities/assets/icon_mvt.png) | 比较页面上元素之间的选件组合，以查看哪个选件组合对特定受众的表现最佳。 此外，在预先指定的整个测试期间，确定该页面的哪个元素最有利于提高转化。<P>有关更多信息，请参阅[多变量测试](/help/main/c-activities/c-multivariate-testing/multivariate-testing.md)。 |
| 体验定位 (XT)<P>![XT图标](/help/main/c-activities/assets/icon_xt.png) | 根据营销人员定义的一套规则和标准向特定受众交付内容。<P>有关更多信息，请参阅[体验定位](/help/main/c-activities/t-experience-target/experience-target.md)。 |

## 为什么要使用此类活动？ {#section_46A70DD7CE3448749E635DDF5EAFC131}

| 活动类型 | 原因 |
|--- |--- |
| 手动 [!UICONTROL A/B测试] | 高度可控的流量测量实验，按百分比而不是按规则进行拆分，让您能够分析测试数据、收集有关受众的分析，并确定性能最佳的体验。 |
| [!UICONTROL 自动分配] | 一种确定入选体验并调整流量分配以尽快将入选体验交付给访客的方式，支持更快捷、更高的转化可能性。 |
| [!UICONTROL 自动定位] | 在多个体验中确定入选者，然后向特定访客提供最适合的体验。随着访客兴趣的不断变化，定位会随着时间的推移而进行调整，因为该算法可以预测访客在特定时间对某种体验的转化倾向。 |
| [!UICONTROL 自当个性化] (AP) | 该方法可个性化一组选件（创建或预定义，在单个页面上或多个页面上的元素中），并交付最能吸引特定访客的选件组合。 |
| [!UICONTROL Multivariate Testing] (MVT) | 一种方法，用于在多个元素中显示多个选件，然后根据特定目标同时测试生成的独特体验，这有助于确定哪个元素变体最成功。 MVT活动还可以揭示哪些元素对访客的交互具有最大的正面或负面影响。 |
| [!UICONTROL 体验定位] (XT) | 一种基于一组定义的分配规则将特定内容定位到特定受众的方法。 |

## 哪类营销人员应该使用活动类型？ {#section_A843D663D3E543FFB1A594266B560395}

| 活动类型 | 营销人员 |
|--- |--- |
| 手动 [!UICONTROL A/B测试] | 熟知相关统计数据知识。<P>具有足够的时间等待测试期结束来分析结果。 |
| [!UICONTROL 自动分配] | 拥有的时间较短。<P>必须确定最佳体验并快速交付。<P>希望能够在测试运行时“偷看”结果。 |
| [!UICONTROL 自动定位] | 拥有几个符合条件的体验。<P>想要根据其动态和不断变化的配置文件，在最佳时间将体验与特定访客匹配。 |
| [!UICONTROL 自当个性化] (AP) | 具有一个或多个选件。<P>想要创建选件组合，以便通过各种唯一的配置文件和行为来为特定访客提供最佳的个性化体验。 |
| [!UICONTROL Multivariate Testing] (MVT) | 熟知相关统计数据知识。<P>具有一个或多个选件。<P>想要分析与页面元素交互相关的转化趋势。 |
| [!UICONTROL 体验定位] (XT) | 必须向特定受众交付特定体验或内容。 |

## 统计详细信息 {#section_22CF2D07DB054505AB5EC702B99A5BB0}

| 活动类型 | 详细信息 |
|--- |--- |
| 手动 [!UICONTROL A/B测试] | 该测试会将每个挑战者体验与控制体验进行比较，然后对所有体验的性能进行排名，确定与控制体验相比较时的入选体验和落选体验。 |
| [!UICONTROL 自动分配] | 该测试能够立即产生真正的入选者并在统计上予以保证，然后将更多流量引导至可能会在该入选体验具有更高转化可能性的受众。 |
| [!UICONTROL 自动定位] | 优化机制会通过显示一段时间内提升度的增加和减少来识别每个体验的相关受众，然后再确定要将哪个体验提供给哪个访客。 优化机制由转化、区段、参数和配置文件脚本通知。 然后，该机制自动选择使用哪种算法来生成更高的提升度和转化率。 |
| [!UICONTROL 自当个性化] (AP) | 根据新访客和过去类似访客的行为，该优化机制会不断调整向哪些访客传递哪些体验，并根据并发的控制组来测量选件的性能。 |
| [!UICONTROL Multivariate Testing] (MVT) | 该测试有助于揭示特定元素对转化的相对影响。 |
| [!UICONTROL 体验定位] (XT) | 该方法定义了将特定体验或特定内容定位到特定受众的规则。客户可以在体验级别进行更新。 |

## 优点和注意事项 {#section_56C46ABEF7B945DDA0C1E6D714377123}

| 活动类型 | 优点 | 注意事项 |
|--- |--- |--- |
| 手动 [!UICONTROL A/B测试] | A/B休息可帮助您全面了解每个体验的表现，而不只是哪个体验的表现最佳。 | 在 [!UICONTROL A/B测试]，如果您在样本量达到之前查看测试结果，则可能会依赖不准确的结果（您不能“偷看”更早！）。<P>取消赞 [!UICONTROL 自动分配]，在A/B测试中，即使在您了解到某些体验优于其他体验之后，流量分配仍保持不变。<P>有关以下项目的最佳实践的信息： [!UICONTROL A/B测试] 活动，请参见 [A/B测试应该持续多长时间](/help/main/c-activities/t-test-ab/sample-size-determination.md) 和 [A/B测试常见的十个隐患以及如何避免它们](/help/main/c-activities/t-test-ab/common-ab-testing-pitfalls.md). |
| [!UICONTROL 自动分配] | [!UICONTROL 自动分配减少了典型 A/B 测试的成本，因为它具有比手动 A/B 测试更高的整体转化率。]转化率较高是因为 [!UICONTROL 自动分配] 将更多流量推送到性能最高的体验，这意味着您可以在测试期结束之前实现该入选体验的好处（您可以偷看！）。 | [!UICONTROL 自动分配会识别入选者，但不区分落选者。]如果您必须了解每个体验的执行方式，则最好使用A/B测试。<P>此 [!UICONTROL 自动分配] 该功能只能与一项高级量度设置配合使用，即“递增计数并保持用户处于活动中”。 如果您不想计算重复的转化次数，则应改用A/B测试。 |
| [!UICONTROL 自动定位] | 替换为 [!UICONTROL 自动定位]，机器学习适用于任何类型的体验，包括多页面体验。 An [!UICONTROL 自动定位] 通过活动，您还可以获得 [!UICONTROL Automated Personalization] 同时使用熟悉的A/B测试工作流。 | 替换为 [!UICONTROL 自动定位]，如果您希望经常或频繁地更改选件内容，则算法需要在每次更改后有足够的时间来利用它学习到的内容并将内容交付给合适的访客。 |
| [!UICONTROL 自当个性化] (AP) | 替换为 [!UICONTROL Automated Personalization]时，您可以在一个位置收集所有选件，算法会确定选件的最佳组合。 您无需指定或构建每一个体验。[!UICONTROL Automated Personalization] 使用与相同的机器学习算法 [!UICONTROL 自动定位]. | 当您合并多个选件时，会发生组合爆炸，导致需要大量的流量。此 [!UICONTROL Automated Personalization] 算法考虑多种因素；因此，需要最多的流量。<P>[!UICONTROL Automated Personalization] 不能在中使用报表 [目标分析](/help/main/c-integrating-target-with-mac/a4t/a4t.md) (A4T)。 |
| [!UICONTROL Multivariate Testing] (MVT) | 替换为 [!UICONTROL Multivariate Testing]中，您可以同时测试多个元素。 | A [!UICONTROL 多变量测试] 耗时，并且由于播放时存在多个变量，因此并不一定能够满怀信心地产生入选的体验。<P>达到完成这种测试所需的流量通常是一项困难的挑战。因为所有 [!UICONTROL 多变量测试] 试验是全因子的，在同一时间可能会有太多的变化元素，从而导致必须测试的多种可能组合。<P>即使是流量相当高的网站也可能无法在可行的时间内完成超过 25 种组合的测试。 |
| [!UICONTROL 体验定位] (XT) | 替换为 [!UICONTROL 体验定位]，您可以根据从任何活动结果中推导出的见解快速执行操作。<P>例如，如果您进行了A/B测试，测试结果表明，挑战体验的表现并未超出控制体验，但与控制体验相比，访客的特定区段在挑战体验上的转化率高出了四倍，那么此时，您便可以使用 [!UICONTROL 体验定位] 将挑战者体验引导至该特定区段。 | [!UICONTROL 体验定位不允许您控制多个受众在一个体验中的比例。] |
