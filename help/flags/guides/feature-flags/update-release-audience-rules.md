# 更新版本受众规则 {#update-release-audience-rules}

## 访问受众设置 {#access}

要开始配置版本接收者，请导航到控制台中的受众设置：

1. 在Flags控制台中打开您的发行版。
2. 转到&#x200B;**受众**&#x200B;选项卡。
3. 在&#x200B;**受众规则**&#x200B;旁启用切换。
4. 选择是添加&#x200B;**包含规则**（谁应接收版本）还是&#x200B;**排除规则**（谁不应接收版本）。

## 支持的受众标准 {#criteria}

### ID type {#id-type}

根据用户的帐户类型（例如，个人或企业帐户）定位用户。

### 国家/地区 {#country}

基于注册国家/地区定位用户。

### 用户 ID {#user-id}

按用户的唯一用户标识符(GUID)定位特定用户。

### 电子邮件地址 {#email}

按用户的确切电子邮件地址定位用户。 若要同时添加多个电子邮件地址，请从运算符下拉列表中选择&#x200B;**in**（而不是&#x200B;**is**），然后：

* 在文本框中输入以逗号分隔的地址列表，或者
* 上载每行具有一个地址的`.txt`文件，或者
* 上载包含地址列表的`.csv`文件

如果列表非常大，请将其分成较小的批次。

### 电子邮件域 {#email-domain}

定位电子邮件地址属于特定域（例如，`yourcompany.com`）的所有用户。

### 客户端IP {#client-ip}

基于客户端IP地址的目标用户。

### 百分比 {#percentage}

以所有用户的随机百分比为目标，包括未经身份验证的用户。

### 百分比（仅经过身份验证） {#percentage-auth}

仅定向经过身份验证的用户的随机百分比。 排除未经身份验证的用户。

## 组合多个规则 {#combining-rules}

您可以使用AND/OR逻辑组合多个受众标准。

**示例：**

* 若要仅定位特定域中的企业帐户，请使用AND将&#x200B;**ID类型**&#x200B;规则与&#x200B;**电子邮件域**&#x200B;规则相结合。
* 若要定位来自特定国家/地区的10%的用户，请将&#x200B;**国家/地区**&#x200B;规则与&#x200B;**百分比**&#x200B;规则相结合。

使用&#x200B;**+/-**&#x200B;图标添加或删除条件。 要复制条件，请选择现有规则旁边的复制图标。 对于复杂的嵌套逻辑，请启用&#x200B;**嵌套逻辑**&#x200B;并在文本框中输入有效的表达式。
