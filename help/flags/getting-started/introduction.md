# 标记简介 {#introduction}

>[!AVAILABILITY]
>
>标记当前位于Beta中，可供Adobe Target客户使用。 请与您的 Adobe 代表联系，请求访问权限。

Adobe Target中的标记是一个受控发布系统，允许团队将功能部署到生产环境，同时精确控制可查看这些功能的人和时间。 一项功能可在生产环境中使用，最终用户不可见，然后针对目标受众逐步开启，而无需任何重新部署。

## 从开发到生产 {#dev-to-prod}

标记支持功能从最早的开发阶段到正式发布的完整历程：

1. **开发人员测试** — 开发人员创建功能标记，将其代码部署到任何环境，并仅针对他们自己的会话进行测试。 没有其他用户受到影响，也不需要分支。

2. **目标预览** — 产品所有者将受众链接到功能标志，从而使该功能可供定义的用户子集（例如，测试版参与者或特定区域）验证和反馈。

3. **逐步推出** — 该功能将逐步向更广泛的受众开放(1%、10%、50%、100%)，并且能够在任何步骤暂停、调整或关闭该功能。

4. **正式发布** — 验证完成后，该功能将可供所有用户使用。

## 核心功能 {#capabilities}

标记是一个功能管理平台，它提供：

* **功能标志** — 在运行时打开或关闭目标受众的任何功能，而无需重新部署代码。

* **受众定位** — 控制哪些人可以使用用户配置文件数据、基于百分比的规则、电子邮件地址、电子邮件域、IP地址或上下文属性查看功能。

* **功能组** — 跨应用程序捆绑多个相关的功能标志，并将它们作为单个单元进行管理，以确保同一受众能够看到一致的体验。

* **版本** — 通过将多个团队和应用程序的功能标志分组到一个版本事件中，协调大型跨团队转出。

* **逐步转出** — 逐步提供阶段功能以降低风险、收集反馈和管理后端负载。

* **删除开关** — 在检测到问题时立即关闭任何功能，无需更改或重新部署代码。

* **AEP SDK支持** — 通过AEP Web SDK和AEP Mobile SDK部署标记，从而支持跨Web和移动应用程序进行一致的标记评估。
