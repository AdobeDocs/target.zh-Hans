# 成功量度

在 [!DNL Adobe Target] 成功量度是用于衡量活动成功与否的参数。 成功量度包括关键业务量度，可让您确定 [!DNL Target] 活动。

例如，您可以确定新的选件是否增加了每个访客带来的收入或是否吸引访客向购物车中添加物品。成功量度可用于发现注册、订购或购买漏斗等方面的问题，也可仅仅用于提高访客或客户参与度。

## 概述

在 [!DNL Target]，则成功量度会预配置最佳选项，以便用于报表和跟踪。

默认情况下，转化事件设置为“[!UICONTROL 递增计数并保持用户处于活动中].&quot; 转化只计数一次，不计数重复的转化，且访客始终会看到活动内容。

设置为“[!UICONTROL 递增计数并保持用户处于活动中]“仅记录同一访客首次订购的订单详细信息。 所有后续订单都会增加转化计数，但不会向RPV/AOV/Sales添加收入，也不会包含在 [!UICONTROL 订单详细信息] 报表。

>[!NOTE]
>
>对于使用 [将Analytics作为报表源](/help/main/c-integrating-target-with-mac/a4t/a4t.md) (A4T)，目标量度将始终使用[!UICONTROL 递增计数并保持用户处于活动中]&quot;和&quot;[!UICONTROL 每次展示时]“ ”设置。 这是 *not* 可配置。

可用的成功量度如下所示：

| 成功量度 | 测量方法 | 定义 |
|--- |--- |--- |
| 转化 | 基于转化 | 转化是指访客在您定义的网站上执行某项操作，例如 <ul><li>单击按钮</li><li>已查看页面</li><li>已完成调查</li><li>购买了</li></ul>转化可以针对每位访客计数一次，也可以针对每位访客完成转化时计数一次。 |
| 收入 | 基于转化 | 由访问带来的收入。您可以选择以下收入量度：<ul><li>每位访客带来的收入 (RPV)</li><li>平均订单值 (AOV)</li><li>销售总额</li><li>订单数</li></ul> |
| 页面查看次数 | 基于参与度 | 将每次独特访问计为一次转化。 |
| 自定义得分 | 基于参与度 | 从访客第一次看到活动的首次显示开始，即基于网站上已访问页面所分配值的汇总分数 [!DNL Target] 请求。 |
| 网站逗留时间 | 基于参与度 | 从访客第一次看到活动显示开始，在访问中逗留的时间（以秒为单位） [!DNL Target] 在会话中请求加载最终页面时发出请求。 |

对于基于参与度的量度（与基于转化和基于收入的量度不同），访客必须在每次访问时重新获得活动参与资格，才会递增该会话的计数。关联的量度将在重新获得资格后开始递增，并在每个访客的会话结束时停止。会话若处于不活动状态超过 30 分钟便会结束。因此，在测试过程中不会立即看到结果；但是，该会话的所有结果在会话结束后的几分钟内都可用。

## 自定义成功量度

您也可以创建自定义成功量度。

选择成功量度后，请选择访客为实现目标所需执行的操作。例如，选择 [!UICONTROL 转化] 量度，将其设置为每位访客计数一次，然后设置当访客查看特定页面（或一组页面）、查看特定页面时是否成功 [!DNL Target] 请求，或单击特定链接。

如果启用，则 [!UICONTROL 一次转化的预计值] 字段(不适用于 [!UICONTROL 页面分数] 量度)会为您的目标提供一个值，但此值不适用于其他量度。 通过此值，[!DNL Target] 可以计算出预计收入提升。此字段为可选字段；但是，如果没有该字段，便无法计算所有非收入量度的增量收入。对于所有收入量度([!UICONTROL 每位访客带来的收入], [!UICONTROL 平均订单值], [!UICONTROL 销售总额]和 [!UICONTROL 订单数])，估计使用 [!UICONTROL 每位访客带来的收入]. 数据类型为货币。请参阅[预计收入提升](/help/main/administrating-target/r-target-account-preferences/estimating-lift-in-revenue.md)以了解更多信息。

查看活动报表时，在报表设置中可以看到您为活动选择的成功量度。

某些量度，例如 [!UICONTROL 自定义评分] 和 [!UICONTROL 每位访客带来的收入]，需要自定义实施以传递订单总数和订单ID等信息。

## 高级设置 {#section_7CE95A2FA8F5438E936C365A6D43BC5B}

可使用高级设置来管理成功的衡量方式。选项包括添加依赖项、选择是将用户保留在活动中还是删除依赖项，以及是对每个参加者还是每次展示都将量度计数一次。

访问 [!UICONTROL 高级设置] 选项，单击 **[!UICONTROL 垂直椭圆]** > **[!UICONTROL 高级设置]**.

![“高级设置”菜单](/help/main/c-activities/r-success-metrics/assets/advanced-settings.png)

>[!NOTE]
>
>如果您将 [!DNL Adobe Analytics] 用作报表源，则设置会由 [!DNL Analytics] 服务器来管理。的 [!UICONTROL 高级设置] 选项将不可用。 有关更多信息，请参阅 [Adobe Analytics作为Adobe Target报表源(A4T)](/help/main/c-integrating-target-with-mac/a4t/a4t.md).

### 添加依赖项

您可以使用高级设置来创建具有依赖关系的成功量度，即仅当访客先到达其他量度时，才会递增一个量度。

![添加依赖项](/help/main/c-activities/r-success-metrics/assets/UI_dep_success_metric.png)

例如，仅当访客在转化之前先点击了选件或访问了某个特定页面时，测试转化才可能有效。

依赖关系功能是 *not* 支持以下内容：

* [!UICONTROL Recommendations] 活动. 其他所有活动类型均支持此功能。
* 如果您使用 [Analytics作为报表源](/help/main/c-integrating-target-with-mac/a4t/a4t.md) (A4T)。
* 量度类型为“已查看页面”。
* 可视化体验编辑器 (VEC) 活动的量度类型为“已单击元素”。

在以下情况下，具有依赖关系的成功量度将无法转化：

* 如果您创建了循坏依赖关系，例如量度 1 依赖于量度 2，而量度 2 又依赖于量度 1，则任何量度都无法转化。
* 实现转化量度后，自动个性化活动会释放用户并重新启动活动，因此，任何依赖于转化量度的量度都将无法转化。

### 用户遇到此目标量度后会出现什么情况？

可使用高级设置来决定用户实现目标量度后将发生的情况。下表显示了可用选项：

| 用户遇到此目标量度后 | 选项 |
|--- |--- |
| [!UICONTROL 增量计数并保持用户处于活动状态] | 指定计数的递增方式：<ul><li>每个参加者一次（默认）</li><li>每次展示时（页面刷新除外）</li><li>每次展示时</li></ul> |
| [!UICONTROL 递增计数、释放用户并允许再次进入] | 选择访客再次进入活动时将看到的体验：<ul><li>相同体验（默认）</li><li>随机体验</li><li>无形体验</li></ul> |
| [!UICONTROL 递增计数、释放用户并阻止再次进入] | 确定用户将看到什么内容来替代活动内容：<ul><li>相同体验（不含“跟踪”功能）（默认）</li><li>默认内容，或其他活动内容</li></ul> |

>[!NOTE]
>
>如果将量度配置为 [!UICONTROL 递增计数] 选项（如上所述），量度计数仅在访客级别为每个参加者正确递增一次。 在访问级别，每个新会话的量度计数每次访问递增一次。

### 如何递增计数：

选择所需的行为：

* 每个参加者一次
* 每次展示时（页面刷新除外）
* 每次展示时
