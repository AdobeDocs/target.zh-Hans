# 限制{#limits}

此信息介绍了字符限制和会影响 Adobe Target 中活动和其他元素的其他限制（选件大小、受众、配置文件、值、参数等）。

下面列出的限制是建议遵循的限制。如果接近或超过这些限制，性能可能会降低。极其复杂的活动（例如一个活动中同时具有多个受众、目标和体验）也可能会导致界面加载速度缓慢。

高度复杂的活动应该先由 Adobe 咨询团队审核，并在限定的环境中进行测试，然后才能发布到生产环境。

## 活动名称

**限制**：250个字符。

## 受众姓名

**限制**：256个字符。

长度超过 256 个字符的值会被截断。

## categoryId 参数

**限制**：250个字符。

## 客户属性名称

**限制**：128个字符。

## 客户属性别名ID

**限制** 50个字符。

## 实体自定义属性

**限制**：最大长度取决于语言。

* 15,000个字符(单值、一个和双字节语言)
* 500个值，每个值100个字符(多个值)

单个值实体自定义属性的最大长度是15,000个字符(对于一个编码的、编码的语言，如英语和其他拉丁语脚本语言)或10,000个字符(对于三字节UTF-8编码语言，如中文、日语和朝鲜语)。

多值实体自定义属性不能包含超过500个值。每个单独的值仅限于100个字符。所有值中的字符总数必须符合单个值实体自定义属性的最大长度限制(请参阅上文)。

## entityID参数

**限制**：1,000个字符。

## ExcludeDID {#excludedid}

**限制**：POST请求KB。2,083字符减去GET请求的URL长度。

对于GET请求，尽管后端的限制为KB，但由于Microsoft Internet Explorer将URL限制为2,083字符，实际限制为2,083字符减去URL的当前长度。

## 体验名称

**限制**：20个字符。

## In-mbox配置文件属性值

**限制**：256个字符。

超过此长度的值会被截断。

## mbox请求中的Mbox内配置文件

**限制**：50个档案。

超过 50 的所有配置文件都会被忽略。

## In-mbox配置文件名称

**限制**：128个字符。

## mbox名称

**限制**：250个字符。

## mbox 参数

**限制**：以下限制适用于mbox参数：

* mbox参数：每mbox500个参数。
* 个人资料参数：500个参数。
* 每个 mbox 的配置文件参数:
* 其他参数（URL、引荐 URL 等）：每个 mbox 50 个其他参数类型。

对于登录到Target数据库的参数，上述限制适用于标准mbox请求。除非因 Web 浏览器限制而缩短请求，否则将应用这些限制。

如果您在 Mobile Services SDK 中使用[批量交付 API](https://developers.adobetarget.com/api/#server-side-batch-delivery)，则该 API 本身的限制为 50 个 mbox 参数、50 个配置文件参数和 50 个其他参数类型。无法使用批量交付 API 发送包含的参数超过这些数量的请求。如果一个请求包含的参数超过这些限制，则该 API 将返回以下错误消息：“mboxParameters 的数量不能超过 100。”

## mbox请求URL

**限制**：2,083个字符。

此限制是由于Microsoft Internet Explorer URL长度限制造成的。

## mbox3rdPartyId 参数

**限制**：60个字符。

## 选件名称

**限制**：250个字符。

## 优惠规模

**限制**：以下大小限制适用于选件：

* HTML 选件为 256 KB.
* UI 中的可视化选件为 64 KB.
* API 中的选件为 512 KB.

如果使用全局mbox，则限制适用于为页面返回的整个内容集。限制选件大小会提升页面加载时间。如果超出限制，则会出现以下消息：

“体验的内容过大，无法发送。请修改体验，以减少受影响的页面代码。”

## orderId 参数

**限制**：120个字符。

建议的限制。

## orderTotal 参数

**限制**：120个字符。

建议的限制。

## productPurchasedId 参数

**限制**：每以逗号分隔的值47个字符。

系统会截断任何内容。

## 可重用受众/帐户

**限制**：75个受众。

建议的限制。如果您拥有太多客户群，则在界面中会出现 JavaScript 超时。

## Target UI中的脚本配置文件输入框

**限制**：2,000个字符。

建议的限制。取决于编码字符串的大小，其大小可能比原始字符串长很多。如果字符串太大，则会在进入 Adobe Target 之前失败。

## 脚本配置文件名称

**限制**：50个字符。

## 脚本配置文件值

**限制**：2,048个字符。

出于性能原因，我们建议返回值不要再超过 256 个字符。

对于字符串返回值，如果返回值的大小超过2,048个字符，则系统将禁用该脚本。

对于数组返回值，如果数组的连接值的大小超过2,048个字符，则系统将禁用该脚本。

## 目标条件

**限制**：1,000个值。

建议的限制。这是指定位文本区域中以行分隔的值的数量。例如，在邮政编码定位中输入 1,000 个邮政编码。
