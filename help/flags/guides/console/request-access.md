# 请求访问 {#request-access}

在团队级别管理对标志的访问。 要使用标志，您需要添加到具有相应角色的团队中。

## 先决条件 {#prerequisites}

在请求访问之前，请确认以下内容：

* 你知道该加入哪一队。 与您的产品或工程主管核实，确定您的团队及其管理员。
* 您知道您需要访问哪个环境 — **暂存**、**生产**，或同时访问两者。

## 请求访问 {#request-steps}

1. 导航到贵组织的访问管理系统。
2. 搜索&#x200B;**标志**&#x200B;并为您的环境（暂存或生产）选择适当的应用程序。
3. 搜索您的团队名称并选择您请求的角色。
4. 提交请求。

提交请求后，您将收到确认。 您可以在访问管理系统中通过&#x200B;**挂起**&#x200B;请求来监视请求的状态。

>[!NOTE]
>
>团队中具有&#x200B;**管理员**&#x200B;或&#x200B;**功能管理员**&#x200B;角色的成员可以批准或拒绝访问请求。

## 可用的角色 {#roles}

分配给您的角色决定了您可以在控制台中执行的操作。 如果您不确定要请求哪个角色，请联系您的团队管理员。

## 授予访问权限后 {#after-access}

您的请求获得批准后，从Adobe Target中打开标志，并验证您的团队和应用程序是否可见。 有关后续步骤，请参阅Adobe Target中的[访问标志](log-in-to-the-console.md)。

## 需要帮助？ {#need-help}

如果您不确定要加入哪个团队，或者您的项目尚无团队，请联系贵组织的标记管理员。
