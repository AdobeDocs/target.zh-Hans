# 为何使用标志 {#why-use}

无论您是在开发环境中测试、使用用户子集进行验证，还是在多个团队中协调大型版本，在需要控制谁查看了某个功能以及何时查看该功能时，标记都是正确的工具。

## 常见用例 {#use-cases}

**开发期间的选择性测试**
针对您自己的会话测试一项功能，而其他开发人员则不受影响地继续工作。 无需进行复杂代码分支。

带有用户反馈的&#x200B;**分阶段转出**
首先向一小部分用户发布功能。 收集反馈，解决问题，然后在完整版本发布之前逐步扩大受众。

**逐步转出到生产环境**
逐步打开新功能 — 依次打开1%、10%、50%和100%的用户。 监视每个步骤的性能和用户响应。 如果发生错误，请立即将其关闭。

**后端负载管理**
逐步分阶段推出以避免后端服务上突然出现流量尖峰，而不是一次让所有用户看到新功能。

**协调的多应用程序版本**
为一组特定用户同时跨多个应用程序和团队启用一项功能。 标记可确保整个发行版表面的一致性。

**延迟的发行版本**
提前将代码部署到生产环境，然后在某个确切的时间点（例如，在产品启动事件开始时）激活该功能，而不用在最后一刻更改任何代码。

**终止开关**
如果在发布后发现问题，则无需修补程序或重新部署即可立即关闭该功能。
