# 标记模式 {#modes}

标记提供两种不同的功能定位模式。 每个版本都有不同的发行控制目的，并且针对特定类型的应用程序或用例而设计。

>[!TIP]
>
>根据您控制的内容选择模式：单个用户会话或组织和环境范围。

## 用户级别定位 {#user-level}

### 概述 {#user-level-overview}

用户级别定位允许在单个用户级别推出功能。 它支持对特定用户、内部测试人员、Canary用户的精确定位以及基于百分比的逐步推出。

此模式最适合需要根据登录者更改体验的应用程序。

### 使用时间 {#user-level-when}

在以下情况下，请使用用户级别定位：

* 为特定用户启用功能
* 运行A/B试验
* 在更广泛的推出之前，先对一小部分人执行Canary测试
* 按用户百分比逐步推出功能
* 根据个别用户属性（如电子邮件域或配置文件数据）优化体验

### 限制 {#user-level-limitations}

用户级定位并非针对以下情况而设计：

* 不控制组织、环境或区域级别的功能
* 不适用于租户范围的功能启用或权利审核

## 组织和环境级别定位 {#org-level}

### 概述 {#org-level-overview}

组织级定位支持在组织、环境和区域范围内推出功能。 它可以跨整个组织或特定部署环境控制功能可用性，而不是针对个别用户。

此模式专为企业级功能审核和特定于环境的转出而设计。

### 使用时间 {#org-level-when}

在执行以下操作时使用组织级别定位：

* 为整个组织启用功能
* 按环境控制功能可用性（例如，暂存与生产）
* 执行区域转出
* 基于权利或订阅层提供功能

### 限制 {#org-level-limitations}

组织级和环境级定位并非针对以下情形而设计：

* 不支持基于丰富用户个人资料的定位
* 不支持在个人用户级别进行基于百分比的转出
* 不用于试验或A/B测试

## 比较 {#comparison}

| 维度 | 用户级别定位 | 组织和环境级别定位 |
|---|---|---|
| 控制范围 | 个人用户 | 组织、环境、地区 |
| 定位基础 | 用户配置文件和属性 | 组织和环境上下文 |
| 百分比转出 | 是 | 否 |
| A/B 测试 | 是 | 否 |
| 企业审核 | 否 | 是 |

## 选择正确的模式 {#choosing}

* 如果您的问题是&#x200B;*“哪些用户应该看到此功能？”* →使用&#x200B;**用户级定位**
* 如果您的问题是&#x200B;*“哪些组织或环境应该具有此功能？”* →使用&#x200B;**组织和环境级定位**
